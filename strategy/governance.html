El Guardián Invisible

En DevOps, no podemos tener un equipo humano revisando cada release manualmente por seguridad. Necesitamos "Automated Governance".

Policy as Code (OPA)

Usamos Open Policy Agent (OPA) para definir reglas que el pipeline debe cumplir antes de desplegar nada.

package kubernetes.admission

deny[msg] {
input.request.kind.kind == "Pod"
image := input.request.object.spec.containers[_].image
not startswith(image, "my-registry.com/")
msg := sprintf("Imagen %v no permitida. Usar registry interno.", [image])
}

Seguridad en el Pipeline (DevSecOps)

  • SAST (Static Application Security Testing): Escanea el código fuente buscando vulnerabilidades conocidas (ej: SQL Injection).
  • SCA (Software Composition Analysis): Revisa tus dependencias (npm, pip, maven) buscando librerías con CVEs conocidos.
  • Secret Scanning: Detecta si alguien subió una API Key o password al repo por error.

Si alguna de estas políticas falla, el pipeline se detiene automáticamente. Calidad es también seguridad.
