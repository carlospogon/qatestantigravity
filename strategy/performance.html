No es solo "Load Testing"

Muchas empresas hacen "load testing" una vez al año antes del Black Friday. La ingeniería de performance es continua. Se trata de entender cómo se comporta el sistema bajo estrés diario.

SLIs y SLOs

Antes de optimizar, debes medir. Definimos:

  • SLI (Service Level Indicator): La métrica real (ej: latencia del 99% de las requests).
  • SLO (Service Level Objective): El objetivo (ej: < 200ms).
  • SLA (Service Level Agreement): El contrato legal (ej: si > 500ms, reembolso).

Continuous Performance Testing con k6

Con herramientas modernas como k6, los tests de performance viven en el código y se ejecutan en el pipeline.

// test.js
import http from 'k6/http';
import { check } from 'k6';

export let options = {
vus: 100, // 100 usuarios virtuales
duration: '30s',
thresholds: {
http_req_duration: ['p(95)<200'], // 95% requests < 200ms
},
};

export default function () {
let res = http.get('https://api.myapp.com');
check(res, { 'status is 200': (r) => r.status === 200 });
}
