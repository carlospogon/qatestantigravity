El mito: "Automatiza todo"

Automatizar el 100% de tus casos de prueba manuales no es una victoria, es una trampa. Los tests UI (Selenium/Cypress) son lentos y frágiles.

La Pirámide de Cohn invertida (El Cono de Helado)

Muchos equipos tienen miles de tests UI y muy pocos unitarios. Resultado:

  • Feedback lento (horas en CI).
  • Muchos "falsos positivos" (tests que fallan por timeout, no por bugs).
  • Desconfianza del equipo en la suite de pruebas.

ROI (Retorno de Inversión)

Antes de automatizar, calcula:

ROI = (Tiempo Manual Ahorrado * Ejecuciones) - (Tiempo Desarrollo Test + Tiempo Mantenimiento Test)

Si el mantenimiento supera al ahorro, mejor hazlo manual o explora testing exploratorio.

La Regla de Oro

Empuja los tests lo más abajo posible en la pirámide. Si puedes testearlo con un test unitario, no hagas un test de API. Si puedes con API, no hagas UI.
